Data flow — dari journal save sampai AI response
1
User saves journal entry
POST /api/journals → Supabase INSERT → Realtime event broadcast
Trigger: journal_entries.INSERT WHERE is_draft=FALSE
2
Edge Function: process-journal fires
Supabase Realtime → Edge Function dipanggil async. Paralel: embedding generation + EI analysis + memory extraction.
Runtime: Deno · Timeout: 60s · Model: GPT-4o-mini (EI) + text-embedding-3-small
3
Embedding generation (async)
Concatenate: main_story + recurring_thoughts + happy_moments → OpenAI text-embedding-3-small → VECTOR(1536) → Upsert journal_embeddings
Cost: ~$0.00002/entry · Skip if content_hash unchanged
4
EI analysis (async)
Journal content → GPT-4o-mini → 7 emotion scores (0.0–1.0) + insight_text + recommended_activity → emotion_analysis table
Cost: ~$0.001/entry · Prompt version tracked for reproducibility
5
Memory extraction (every 3rd journal)
Journal content → GPT-4o-mini → fact/pattern/preference extraction → ai_user_memory (max 3 facts/session) + embedding per fact
Condition: total_entries % 3 === 0 · Confidence threshold: >0.6
6
AI Coach context assembly (on demand)
Paralel fetch: today's journal + semantic memory search (top-5 by cosine) + similar past journals (top-3) + last 10 chat messages + user profile
Total context: ~2000–2500 tokens · Budget per source defined in prompts.ts
7
GPT-4o streaming response
Assembled context + user message → GPT-4o (streaming) → SSE chunks → client → bubble expand animation. Save to coach_messages.
Model: GPT-4o · Max tokens: 500 · Temperature: 0.8 · stream: true
8
Dashboard update (Realtime)
emotion_analysis → Supabase Realtime → client subscription → dashboard AI insight widget updates tanpa refresh halaman
Latency target: insight visible dalam 60s setelah journal save

Emotional Intelligence Engine — 7 dimensi deteksi
Stress
Deteksi: keluhan beban kerja, deadline, konflik. Kata kunci: "capek", "berat", "overwhelmed".
Anxiety
Deteksi: kekhawatiran masa depan, overthinking, ketidakpastian. Kata: "khawatir", "bagaimana jika".
Gratitude
Deteksi: ekspresi syukur, apresiasi positif. Kata: "bersyukur", "senang karena", "beruntung".
Burnout
Deteksi: kelelahan emosional + fisik berkelanjutan. Sinyal: "tidak ada energi", "tidak peduli lagi".
Overthinking
Deteksi: pikiran berputar, rumination, hyperanalysis. Pola: repetisi topik, pertanyaan tanpa jawaban.
Optimism
Deteksi: orientasi masa depan positif, harapan, rencana. Kata: "akan", "berencana", "semoga".
Self-compassion
Deteksi: perlakuan diri sendiri dengan kebaikan, self-forgiveness. Kata: "aku maafkan diri", "wajar".
Action triggers berdasarkan score
stress > 0.8 → Sarankan breathing exercise + gentle professional referral di AI Coach response
burnout > 0.7 → Rekomendasikan rest day content + notifikasi reminder besok lebih gentle
gratitude > 0.7 → Simpan ke memory: "pengguna memiliki kemampuan syukur yang kuat"
overthinking > 0.7 → AI Coach menggunakan pertanyaan grounding: "apa yang bisa kamu kontrol sekarang?"
RAG Architecture & Long-term Memory System
Embedding Layer — text-embedding-3-small (1536 dim)
journal_entries → content hash + embedding coach_messages → per message embedding ai_user_memory → per fact embedding
Vector Search — pgvector IVFFlat cosine similarity
journal_embeddings (IVFFlat lists=100) coach_messages (IVFFlat lists=50) ai_user_memory (IVFFlat lists=50)
Upgrade ke HNSW saat rows > 1M. Threshold cosine similarity: 0.75
Long-term Memory Types (ai_user_memory)
fact "Asmawati adalah seorang guru SMA dengan dua anak"
pattern "Sering merasa cemas dan kewalahan setiap Senin pagi"
preference "Lebih nyaman menulis jurnal di malam hari setelah anak tidur"
trigger "Stres meningkat saat deadline mendekat atau saat anak sakit"
achievement "Mencapai streak 30 hari pertama pada Mei 2025"
Memory Lifecycle & Quality Control
Extract setiap 3 jurnal (biaya optimal) · Confidence threshold 0.6 · Max 3 memories per extraction · reference_count naik setiap AI gunakan memory tersebut · Memory dengan confidence < 0.4 setelah 30 hari tanpa reference dinonaktifkan · is_active = FALSE tidak dihapus (audit trail)
Semantic search function
search_journals_semantic(user_id, embedding, limit=5, threshold=0.75)
→ IVFFlat cosine search → top-5 entries → return {entry_id, entry_date, similarity, snippet}
Insight Generator — 4 tier temporal insights
Daily insight
Triggered after journal save
Dari EI analysis 1 hari. Model: GPT-4o-mini. Output: title + body + emoji + stat. Contoh: "Mood +18% vs kemarin. Hari yang produktif!"
Weekly insight
Setiap Minggu malam
Agregat mv_weekly_user_stats. Pattern detection lintas 7 hari. Contoh: "Kamu lebih bahagia di hari kerja minggu ini."
Monthly insight
Hari pertama bulan baru
Agregat mv_monthly_user_stats + memory vault. Deep pattern analysis. Contoh: "72% hari bahagiamu terjadi saat bersama keluarga."
Yearly insight
1 Januari setiap tahun
Full year analysis + annual_reviews generation. AI menulis "judul tahun" personal. Contoh: "2025 — Tahun Keberanian Kecil".

Recommendation Engine — kontekstual dan berbasis data
Rule-based triggers (cepat, tanpa AI call)
stress > 0.8 → Rekomendasikan: ruang bernapas 4-4-4-4, soundscape hujan
burnout > 0.7 → Rekomendasikan: pengingat lebih gentle besok, konten self-care
streak risk → Push notif jam 20:00 dengan pesan streak personal
habit_corr > 0.4 → Tampilkan insight korelasi: "Hari olahraga, mood kamu +18%"
mood < 4 × 3 → Sarankan life wheel check-in — mungkin ada area yang perlu perhatian
AI-generated recommendations (GPT-4o-mini, weekly)
Input: mv_weekly_user_stats + habit_correlations + ai_user_memory (patterns) → GPT-4o-mini → 3 rekomendasi personal untuk minggu depan. Format: {activity, reason, estimated_impact}. Contoh: "Coba olahraga Senin pagi — data menunjukkan mood kamu 18% lebih baik di hari olahraga."

Personalization Engine — adaptive user model
Tier 1 — Explicit
Data dari pengguna langsung
Onboarding answers, user_preferences, notification settings, journal template choice, AI coach ratings (was_helpful)
Tier 2 — Behavioral
Observasi pola penggunaan
Jam menulis, langkah yang sering dilewati, durasi per session, fitur yang paling sering dipakai, notification open rate
Tier 3 — AI-inferred
Patterns dari konten jurnal
Emotional patterns, recurring themes, stress triggers, gratitude topics, growth areas dari ai_user_memory dan EI analysis
Personalization yang diaplikasikan
Sapaan AI: Bloom menyebut nama, referensi orang dan situasi yang dikenal dari memori
Journal prompts: Pertanyaan disesuaikan dengan topik yang sering muncul di jurnal sebelumnya
Notification timing: Dikirim berdasarkan jam pengguna paling sering menulis, bukan jam default
Insight framing: Tone insight disesuaikan: lebih supportive untuk high-stress users, lebih challenging untuk growth-oriented users
Afirmasi preset: Afirmasi yang ditampilkan di step 14 disesuaikan dengan mood_category hari ini dan emotional pattern mingguan
